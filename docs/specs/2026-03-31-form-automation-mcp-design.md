# 폼 자동화 MCP 서버 설계 문서

## 개요

기획자가 매일 반복하는 내부 시스템 폼 입력 업무를 자동화하는 MCP 서버.
Claude Desktop에서 자연어로 요청하면 Playwright가 브라우저를 띄워 메뉴 탐색 → 폼 입력 → 저장을 자동 수행한다.

## 배경

- 기획자가 매일 10건 이상 내부 시스템에 폼을 반복 입력 중
- 메뉴 클릭 → 페이지 이동 → 텍스트 입력 → 저장의 인터랙티브한 흐름
- 비개발자 2-3명이 사용, Mac/Windows 혼합 환경
- 로그인은 미리 해둔 상태에서 사용

## 기술 스택

- **Node.js** — MCP 서버 런타임
- **@modelcontextprotocol/sdk** — MCP 프로토콜 구현
- **Playwright** — 브라우저 자동화
- **xlsx (SheetJS)** — 엑셀/CSV 파일 읽기
- **googleapis** — Google Sheets API 연동

## 아키텍처

```
사용자 (Claude Desktop)
    │ "이 엑셀 파일로 폼 입력해줘"
    ▼
MCP Server (로컬 Node.js 프로세스)
    ├── 데이터 소스 리더
    │   ├── 엑셀/CSV 리더 (xlsx)
    │   └── Google Sheets 리더 (googleapis)
    │
    ├── 데이터 정규화
    │   └── 어떤 소스든 동일한 배열 형태로 변환
    │       [ { 메뉴경로: "...", 라벨1: "값1", 라벨2: "값2" }, ... ]
    │
    ├── 워크플로우 실행기 (Playwright)
    │   └── 행마다 반복:
    │       1. 메뉴 경로 파싱 → 사이드 메뉴 클릭 (하위 메뉴 펼침)
    │       2. 대상 페이지 로드 대기
    │       3. 라벨 기반 필드 매칭 → 값 입력
    │       4. 저장 버튼 클릭
    │       5. 저장 완료 확인
    │
    └── 결과 리포터
        └── 성공/실패 건수 + 실패 상세 (스크린샷 포함)
```

각 PC에서 로컬로 실행되며, 별도 서버 인프라 불필요.
Claude Desktop이 실행 시 자동으로 MCP 서버 프로세스를 띄움.

## 데이터 입력 소스

### 엑셀/CSV

파일을 Claude Desktop에 전달하면 MCP 서버가 읽음.

엑셀 구조 규칙:
- **첫 번째 열**: `메뉴경로` (예: `프로젝트 > 일정관리`)
- **나머지 열**: 열 이름 = 폼의 라벨 텍스트

예시:

| 메뉴경로 | 프로젝트명 | 설명 | 비고 |
|---------|----------|------|------|
| 프로젝트 > 일정관리 | 3월 리뷰 | 주간보고 작성 | 완료 |
| 프로젝트 > 산출물 | 기획서 v2 | 최종본 업로드 | - |
| 운영 > 공지사항 | 4월 일정 | 휴무일 안내 | 전체공지 |

### Google Sheets

시트 URL을 전달하면 Google Sheets API로 읽음.
엑셀과 동일한 구조 규칙 적용.

초기 세팅 시 Google API 키 발급 + 시트 공유 설정 필요 (1회).

## MCP 도구 정의

### `run-workflow`

메인 도구. 데이터 소스에서 읽은 데이터를 기반으로 폼 입력 워크플로우 실행.

**파라미터:**
- `source_type`: `"excel"` | `"google_sheets"`
- `file_path`: 엑셀/CSV 파일 경로 (source_type이 excel일 때)
- `sheet_url`: Google Sheets URL (source_type이 google_sheets일 때)
- `site_url`: 내부 시스템 URL
- `mode`: `"visible"` (기본) | `"fast"`

**반환:**
- 총 처리 건수, 성공 건수, 실패 건수
- 실패 건별 상세 (행 번호, 메뉴경로, 에러 메시지, 스크린샷 경로)

## 브라우저 자동화 상세

### 메뉴 네비게이션

메뉴경로를 `>` 기준으로 분리하여 순차적으로 클릭:
```
"프로젝트 > 일정관리"
→ 1) "프로젝트" 메뉴 클릭 (하위 메뉴 펼침)
→ 2) "일정관리" 클릭
→ 3) 페이지 로드 대기
```

### 라벨 기반 필드 입력

Playwright의 `getByLabel()` 사용:
```javascript
// 엑셀 열 이름이 "프로젝트명"이면
await page.getByLabel('프로젝트명').fill('3월 리뷰');
```

엑셀 열 이름과 화면 라벨 텍스트를 일치시키는 것이 핵심.

### 저장

각 폼 입력 완료 후 저장 버튼 클릭.
저장 성공/실패 여부를 확인한 후 다음 행으로 진행.

## 실행 모드

| 모드 | 동작 | 용도 |
|------|------|------|
| **보기 모드** (기본) | 브라우저가 화면에 표시됨, 스텝 간 0.5~1초 딜레이 | 과정 확인, 신뢰 확보 |
| **빠른 모드** | 백그라운드 실행, 딜레이 없음 | 익숙해진 후 빠르게 처리 |

## 에러 처리

| 상황 | 처리 방식 |
|------|----------|
| 메뉴를 못 찾음 | 해당 행 스킵, 에러 로그 기록 후 다음 행 계속 |
| 라벨에 해당하는 필드 없음 | 해당 필드 스킵, 경고 로그 기록 |
| 저장 실패 (유효성 검증 등) | 스크린샷 캡처 + 에러 메시지 기록 |
| 페이지 로드 타임아웃 | 1회 재시도 → 실패 시 스킵 |
| 세션 만료 (로그인 풀림) | 자동화 중단, 현재까지 결과 반환 |

핵심 원칙: 한 건 실패해도 나머지는 계속 진행.

## 결과 리포트

실행 완료 후 Claude Desktop 대화에 반환:

```
완료: 8건 / 실패: 2건

성공:
 1행 - 프로젝트 > 일정관리 → 완료
 2행 - 프로젝트 > 산출물 → 완료
 ...

실패:
 4행 - 운영 > 공지사항 → "공지유형" 라벨을 찾을 수 없음
 9행 - 프로젝트 > 일정관리 → 저장 실패 (스크린샷: errors/row9.png)
```

## 배포 및 설치

### 레포지토리

GitHub `jayounglee92` 계정에 새 레포지토리 생성하여 코드 관리.

### 초기 세팅 (개발자가 1회 수행)

1. 각 PC에 Node.js 설치
2. 레포지토리 클론: `git clone <repo-url>`
3. 의존성 설치: `npm install`
4. Claude Desktop 설정 파일(`claude_desktop_config.json`)에 MCP 서버 추가:
   ```json
   {
     "mcpServers": {
       "form-automation": {
         "command": "node",
         "args": ["<경로>/dist/index.js"],
         "env": {
           "SITE_URL": "https://internal-system.example.com"
         }
       }
     }
   }
   ```
5. (Google Sheets 사용 시) Google API 키 발급 + 환경변수 설정

### 이후 사용

기획자는 Claude Desktop만 열면 바로 사용 가능:
- "이 엑셀 파일로 폼 입력해줘"
- "이 구글 시트로 폼 입력해줘"
- "빠르게 폼 입력해줘"

## 엑셀 템플릿

프로젝트에 샘플 엑셀 템플릿 포함:
- 열 이름이 폼 라벨과 일치하도록 미리 세팅
- 기획자가 복사하여 데이터만 채우면 됨

## 설정 파일

사이트별 설정을 별도 config 파일로 관리:
```json
{
  "siteUrl": "https://internal-system.example.com",
  "menuSelector": ".side-menu",
  "saveButtonText": "저장",
  "defaultMode": "visible",
  "stepDelay": 500
}
```

내부 시스템 UI가 변경되면 이 설정 파일만 수정하면 됨.
